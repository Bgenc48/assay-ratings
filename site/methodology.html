Methodology, version 0.1

Published in full, versioned, and applied mechanically. The grade measures whether a token can betray you in ways it hasn't disclosed — not whether it is a good investment. Changes ship with a public changelog entry; every published score is stamped with the version that produced it.

Three consequences, all deliberate

  • Disclosed, purpose-consistent centralization is not a lie. A regulated fiat stablecoin is pausable and blacklistable by design and says so. It loses points for the raw power surface; it is never treated like a token hiding a mint backdoor. Assay grades the gap between representation and reality plus structural exposure — not centralization per se.
  • A memecoin that promises nothing and hides nothing can outscore a “serious” project that lies. B means “you will not be robbed by mechanism,” not “number go up.”
  • Trust needs time. Hard age caps mean no token gets an A in its first year — including any token affiliated with Assay's operator, which is not graded at all (COI policy).

The eight dimensions

Supply and admin weigh most because they are the two vectors that can take 100% of holder value in one transaction. Concentration weighs least because address labeling is the noisiest data — its extremes are handled by hard caps instead. Missing data always surfaces as reduced coverage (●/◐/○), never as a fabricated score.

Verdicts on claims

Assay extracts the machine-checkable claims a project makes in its own material — each recorded with a verbatim quote and source URL — and verifies each against chain state on every scan: VERIFIED, FALSE, UNVERIFIABLE, STALE, or FORWARD-LOOKING. Present-tense claims (“liquidity is locked”) are checked now; forward-looking promises become tracked commitments with deadlines and turn FALSE only if the deadline passes unmet. Undisclosed on-chain powers count as omissions; honest self-disclosure of warts scores up. Off-chain claims (team, partnerships, “backed by”) are marked UNVERIFIABLE — never verified, never contradicted.

Integrity rule: language models help extract claims from documents; they never produce verdicts and never touch the score. Every verdict requires a deterministic on-chain check, and extracted claims are inert until a human review approves them into the public registry. Docs are treated strictly as untrusted data — text inside them cannot instruct the pipeline.

Grades and hard caps

Scale: A+, A, A−, B+, B, B−, C, D, F, plus N/R (unratable) and a Provisional badge under 12 months. Numeric bands: A ≥ 85 (A+ ≥ 95), B 70–84, C 55–69, D 40–54, F < 40. The overall grade is min(weighted average, hard caps) — a pure average would let beautiful documentation buy back a mint backdoor; pure weakest-link would give projects no gradient to climb.

Trust Model badge

A B+ from a disclosed custodial stablecoin and a B+ from a young fixed-supply token are different kinds of B+. Every grade carries a badge saying which kind of trust you are extending: Code-Enforced (no admin exists), Governance-Gated (powers behind binding on-chain governance and timelocks), Custodial (powers behind a multisig; “disclosed” when the docs enumerate them honestly), or Discretionary (someone can act unilaterally).

Data sources & honest limits

  • Chain state via public JSON-RPC endpoints (every batch verifies the endpoint's chain ID); holder and contract data via Blockscout; pool discovery via DexScreener's public API. No Etherscan API anywhere — its terms prohibit commercial and AI/dataset use.
  • v0.1 analyzes classic (v2-style) pools' LP distribution on-chain; concentrated-liquidity (v3/v4) positions show as “insufficient data”, never as “safe”. Locker credit requires the locker to be on the public, human-reviewed allowlist.
  • Insider float and vesting compliance are only automatable when vesting is on-chain; custodial “locks” are UNVERIFIABLE by design.
  • Audit claims are scored link-resolves-only. Assay does not judge auditor quality.
  • Low grades (D/F) on tokens above $1M liquidity are human-reviewed before publication.
